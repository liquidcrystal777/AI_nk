# 屏幕流程与导航设计要求

按照用户提供的详细线框图设计页面和导航。

## 1. 全局导航规则
- **设置页**：在所有页面的顶部导航栏增加一个 Lucide React 的齿轮图标，点击跳到 /settings（让用户填写 AI Key 和 Base URL）。
- **返回**：大多数页面顶部应包含一个 Lucide React 的向左箭头，点击返回上一级或 Home Screen。

## 2. Home Screen (主页)
- **视觉**：显示巨大的词库总词汇量数字 (如 "2000")。下方三个扁平化的清华紫圆角按钮：浏览词库 (browse)、录入新词 (record)、开始复习 (review)。
- **导航**：点击按钮分别跳转到对应页面。

## 3. Browse Screen (浏览词库页)
- **视觉**：顶部带有 Lucide React 的搜索图标。单词列表卡片扁平风，点击卡片展开显示考点例句列表。支持左滑跳词。
- **导航**：箭头指示：右滑可以回到主界面。点击卡片跳转查看详情。

## 4. Record Screen (录入新词流)
这是一个多步流，由用户手动输入提示词和 AI 生成内容组成。
- **Record Screen(1) (初始录入页)**：
  - **视觉**：包含一个清晰的输入框（这个是输入提示词）。下方两个选择按钮（choose year、choose text）。
  - **导航**：点击顶部导航栏的 Lucide 确认图标或底部的“确认”按钮（名字需对齐，如 "生成卡片"）提交给 AI。提交后加载并跳转到 Record Screen(2)。
- **Record Screen(2) (结果编辑页)**：
  - **视觉**：展示 AI 生成的单词卡片（单词、音标、释义、例句数组）。**所有字段都是可被用户手动编辑的输入框**。
- **Record 页流程导航**：用户编辑完卡片后，点击顶部的“打钩图标”（确认键），正式将单词存入 Dexie 数据库，存入后**清除输入并跳回 Home Screen**。

## 5. Review Screen (开始复习流)
- **Review Screen(1) (核心流转页)**：
  - **视觉**：巨大、极粗的单词拼写 (`text-5xl font-black tracking-tight`)。下方是四个圆形、功能明确的Lucide图标按钮：减号 (-), 圆圈 (⚪), 加号 (+), 带箭头的按钮。
  - **导航**：按钮点击触发状态流转（详见 03 文档）。
