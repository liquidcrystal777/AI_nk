# 复习状态机逻辑

核心复习页 (`/review`) 展示 `nextReviewTime` 最小的单词。

## 核心状态流转规则
用户点击下方四个按钮，立即触发状态跃迁和 `nextReviewTime` 计算：

| 按钮 (图示图标) | 对应主观判断 | 对应状态动作 | 状态跃迁示例 | 新 nextReviewTime |
| :--- | :--- | :--- | :--- | :--- |
| **`-`** | 完全不认识 / 降级 | **降级/遗忘 (Failure)** | `known` -> `vague`, `mastered` -> `vague` | 5分钟后 |
| **`⚪`** | 模糊 / 需巩固 | **保持/巩固 (Maintain)** | `new` -> `new` (标记为已背过), `vague` -> `vague` (重新排序), `known` -> `known` (巩固) | 20分钟后 |
| **`+`** | 认识 / 升级 | **认识/掌握 (Success)** | `new` -> `known`, `vague` -> `known`, `known` -> `mastered` | `new`/`vague` -> 12小时后；`known` -> 3天后 |
| **带有箭头的按钮** | 跳过 / 下一个词 | **跳过 (Skip)** | 状态不改变 | 不改变排序（直接拉取下一个词） |

## 拼写判断子模式 (简化理解):
虽然线框图显示的 Review Screen(2) 作为一个四选一的判断页面。为了保持核心流转逻辑清晰，我们先将 Review Screen(2) 的拼写判断暂时简化为对核心四按钮逻辑的一种视觉补充（例如，当点击 `-` 按钮时，可以先让单词拼写变淡，让用户脑补，脑补失败按`-`，脑补成功按`+`）。核心流转依然由四个主按钮驱动。

每次点击后，立即调用 `db.words.update()` 更新 IndexedDB。
